BEGINNER NOTES 

1: Variables and Data Types

· Variables: Named storage locations in memory that hold values. They must be declared with a type before use.
· Declaration: int age; (creates variable without value).

· Initialization: age = 25; or int age = 25; (assigns first value).

· Data Types: Define the kind of data a variable can hold (e.g., int, float, char, bool).

· Memory Allocation: Compiler reserves specific bytes based on data type (e.g., int → 4 bytes).

· Scope: Region where variable is accessible (local inside {}, global outside functions).

 2: Fibonacci Series Program

Iterative method (efficient):

```python
a, b = 0, 1
for i in range(n):
    print(a)
    a, b = b, a + b
```

Recursive method (simple but slower):

```python
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)
```

· Fibonacci sequence: 0, 1, 1, 2, 3, 5, 8...
· Mathematical series: F(n) = F(n-1) + F(n-2).
· Algorithm: Uses loops (iteration) or function recursion to generate terms.

4: Bubble Sort Algorithm

```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n-1):                # Outer loop: passes
        for j in range(0, n-i-1):       # Inner loop: comparisons
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]   # swap
    return arr
```


· Sorting technique: Repeatedly steps through list, compares adjacent elements, swaps if in wrong order.

· Comparison: Compares arr[j] and arr[j+1].

· Nested loops: Outer for passes, inner for comparisons.

· Time complexity: O(n²) worst/average case, O(n) best case (already sorted).

1: Pointers in C/C++

· Pointers: Variables that store memory addresses of other variables.
·
 Declaration: int *ptr; (pointer to an integer).

· Dereferencing: *ptr accesses the value at the stored address.

· Pointer arithmetic: Adding/subtracting moves pointer by sizeof(type) bytes (e.g., ptr++ moves to next integer).

· Memory management: Used for dynamic allocation (malloc, new) and direct hardware access.

· Dangerous programming: Can cause segmentation faults, memory leaks, dangling pointers, and buffer overflows if mishandled.

 2: Stack vs Queue

Feature Stack Queue

Stack

· Principle: LIFO (Last In, First Out)
· Operations: push (add to top), pop (remove from top)· Access: Only the top element is accessible
· Real-world examples: Undo/redo feature, browser back button, function call stack, plate pile
· Applications: Expression evaluation, backtracking algorithms, syntax parsing

Queue

· Principle: FIFO (First In, First Out)
· Operations: enqueue (add to rear), dequeue (remove from front)
· Access: Both front and rear ends are used
· Real-world examples: Printer queue, ticket counter line, customer service queue, BFS graph traversal
· Applications: Task scheduling, message buffering, breadth-first search, resource sharing

Implementation: Both can be implemented using arrays or linked lists. Stack uses a single pointer (top), while Queue requires two pointers (front and rear).

 3: Binary Search in Sorted Array

Iterative implementation (Python):

```python
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```

· Divide and conquer: Repeatedly split search interval in half.
· Logarithmic time: O(log n) time complexity.
· Prerequisite: Array must be sorted.
· Search algorithm: Compares middle element, then searches left or right half.
· Efficiency: Much faster than linear search for large arrays.

 4: Object-Oriented Programming Concepts

· OOP: Programming paradigm based on real-world modeling using objects that contain data and behavior.

· Class: Blueprint/template for creating objects (e.g., class Car { }).

· Object: Instance of a class with actual values (e.g., Car myCar = new Car()).

ADVANCED NOTES

 1: Time and Space Complexity with Big O Notation

· Big O notation: Mathematical framework to describe upper bound of algorithm growth rate as input size → infinity. Ignores constants and lower-order terms.

· Time complexity: Measures number of operations an algorithm executes relative to input size (n).

· Space complexity: Measures extra memory an algorithm requires relative to input size.

· Algorithm analysis: Used to compare efficiency independent of hardware/language.

· Efficiency comparison (fastest to slowest):

  · O(1) – Constant (array access)
  · O(log n) – Logarithmic (binary search)
  · O(n) – Linear (linear search)
  · O(n log n) – Linearithmic (merge sort)
  · O(n²) – Quadratic (bubble sort)
  · O(2ⁿ) – Exponential (recursive Fibonacci)

· Asymptotic behavior: Describes how algorithm performs for very large inputs, where growth rate dominates constant factors.

2: Graph Traversal Algorithms (BFS and DFS)

BFS (Breadth-First Search) – Iterative using queue:

```python
def bfs(graph, start):
    visited = set()
    queue = [start]
    visited.add(start)
    while queue:
        node = queue.pop(0)
        print(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
```

DFS (Depth-First Search) – Recursive or stack:

```python
def dfs(graph, node, visited):
    visited.add(node)
    print(node)
    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs(graph, neighbor, visited)
```

· Graph traversal: Visiting all nodes in a graph systematically.
· Adjacency list: Common representation using dictionary/hash map.
· Visited nodes: Track to avoid cycles and infinite loops.
· Path finding: Both used in shortest path (BFS for unweighted), connected components.
· Network algorithms: Social networks, web crawling, GPS navigation.

Difference: BFS explores level-by-level (queue), DFS explores depth-first (stack/recursion).

3: Hash Tables and Collision Handling

· Hash tables: Data structure storing key-value pairs using a hash function to compute index into an array.

· Hash functions: Convert key into integer index (e.g., hash(key) % table_size). Goal: uniform distribution.

· Collision resolution: When two keys hash to same index.

· Chaining: Each bucket contains a linked list of entries. Simple, handles many collisions, but can degrade to O(n).

· Open addressing: Find next available slot (linear probing: index+1, quadratic probing, double hashing). Saves memory but requires careful deletion.

· Load factor: α = n / m (entries / table size). Threshold triggers rehashing (resize + reinsert all keys). Typical threshold: 0.7–0.75.

· Performance optimization: Good hash function + appropriate load factor + resize strategy gives average O(1) operations.

4: Balanced Binary Search Tree

BST (Binary Search Tree) : For each node, left subtree < node < right subtree. Worst-case O(n) if skewed.

Balanced tree: Maintains height O(log n) for all operations.

AVL Tree (first self-balancing):

· Each node stores height difference of left/right subtrees (balance factor: -1, 0, or +1).

· Rotation operations: Restore balance after insertion/deletion
 
 · Right rotation (Left-Left case)

  · Left rotation (Right-Right case)
 
 · Left-Right rotation (rotate left child left, then right)

  · Right-Left rotation (rotate right child right, then left)

Red-Black Tree (more relaxed):

· Nodes colored red/black with 5 rules; fewer rotations than AVL.

Self-balancing: Automatically performs rotations during insert/delete to keep height O(log n).

Search efficiency: O(log n) for search, insert, delete in worst case (vs O(n) for unbalanced BST).
